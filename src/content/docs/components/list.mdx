El componente **List** `<ui-list>` es la unidad para construir listas verticales, menús de navegación o paneles de configuración.

Diseñado para presentar datos de manera lineal y compacta. El **List** está optimizado para la repetición y la navegación lineal.

**Vista previa del componente**

<div class="components-structure">
    <div class="responsive-structure-full"> 
        <ui-list text="Item" description="Description">
            <svg slot="end-element" width="24" height="24" viewBox="0 0 24 24" fill="none" xmlns="http://www.w3.org/2000/svg">
                <path d="M9 18L15 12L9 6" stroke="currentColor" stroke-width="1.6" stroke-linecap="round" stroke-linejoin="round"/>
            </svg>
        </ui-list>
    </div>
</div>

## ¿Cuándo debería usarse?
Usa este componente cuando quieras:

- Menús de Configuración: Listas de opciones donde cada fila lleva a una pantalla de detalle o ajuste _(ej. Ajustes del perfil)_.
- Navegación Lateral: Elementos de un drawer o menú principal.
- Listas de Selección: Cuando necesitas mostrar múltiples opciones seleccionables de forma vertical.

## Uso Básico del Componente
El componente acepta atributos simples para el contenido textual _(text y description)_ y utiliza **Slots Nombrados** para los elementos visuales al inicio o al final.

## Variantes
La versatilidad del componente `<ui-list>` radica en su capacidad para combinar información textual con elementos visuales de apoyo. Su estructura se divide en tres áreas clave:

- **Contenido Principal (Texto):** El núcleo del componente, definido por atributos.
- **Elemento de Inicio (Slot):** Generalmente reservado para iconos o avatares que categorizan la entrada.
- **Elemento Final (Slot):** Reservado para iconos de navegación, interruptores, o metadatos.

## Disparador de Eventos
Cuando el usuario hace clic en un ítem de la lista que no está deshabilitado, el componente **emite** el Evento Personalizado `list-item-click`.

Si el atributo `disabled` está presente, el componente ignora los clics y no emite el evento, protegiendo la lógica de tu aplicación de interacciones no deseadas.

### Estructura de `detail`
El evento entrega un objeto detail con el identificador único del elemento seleccionado.

| Propiedad | Tipo   | Descripción |
|-----------|--------|-------------|
| `id`      | **string** | ID único del ítem. Es indispensable para identificar qué opción específica de la lista fue seleccionada. |

<Aside type="caution" title='Importante'> Por lo tanto, es una **práctica obligatoria** asignar manualmente el atributo `id` a todas las instancias de los inputs para saber identificarlos</Aside>

## Obtención del evento
Para escuchar los eventos emitidos por los componentes de este UI Kit, la opción más confiable es usar **`addEventListener`**, ya que garantiza que los eventos personalizados se reciban **correctamente**.

En **frameworks** _(Vue.js, React, Angular)_ también puede usarse **`addEventListener`**, aunque cada uno ofrece su propia forma de manejar eventos, que igualmente es válida y en la mayoría de los casos funcionan sin problema. Sin embargo, cada framework tiene particularidades importantes que tu como desarrollador debes considerar:

- **Vue**: Normalmente funciona de forma nativa, ya que Vue respeta los **eventos del DOM** tal como se emiten.
- **Angular**: Suele funcionar, en algunas versiones antiguas del framework tenían **limitaciones** con eventos provenientes de Web Components.
- **React**: Puede detectar eventos personalizados pero no siempre. Su sistema interno de `Synthetic Events` no garantiza la **compatibilidad al 100%**, por lo que el comportamiento puede variar entre navegadores y versiones.
