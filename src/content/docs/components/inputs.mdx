El componente **Input** `ui-input` es el campo estándar para la recolección de datos de formato libre del usuario (texto, números, etc.). Este componente centraliza la gestión de toda la experiencia de entrada: etiquetas, descripciones, íconos y estados de error. 

Proporcionar una experiencia de entrada de datos pre-estilizada y validada que se adapta a las reglas de tu sistema.

## ¿Cuándo debería usarse?
Usa este componente cuando quieras:

- Recoger texto, correo electrónico o contraseñas.
- Guiar al usuario con un label y una descripción clara.

## Uso básico del componente

El componente se integra directamente en el DOM. Para renderizarlo en el sistema, utiliza la siguiente etiqueta:

export const exampleCode = `<ui-input></ui-input>`;
export const fileName = 'ejemplo.html';

## Disparador de Eventos
El componente `<ui-input>` es un proxy para los eventos del campo de entrada nativo. **Re-emite** dos eventos **clave** para la gestión de datos, ambos como Eventos Personalizados ***(CustomEvent)***:

- `input`: Se dispara en cada pulsación de tecla, ideal para binding instantáneo ***(ej. autocompletar, previsualización de datos)***.
- `change`: Se dispara cuando el usuario **finaliza la entrada** y el campo pierde el foco _(blur)_, ideal para validaciones finales de formulario.


### Estructura de `detail`
Ambos eventos ***(input y change)*** emiten un objeto detail, manteniendo la consistencia:

| Propiedad | Tipo          | Descripción                                                                              |
|-----------|---------------|--------------------------------------------------------------------------------------------|
| `value`   | <span style={{whiteSpace: "nowrap"}}>`string` </span> | El contenido actual del campo de texto.                   |
| `id`      | <span style={{whiteSpace: "nowrap"}}>`string` </span>     | El ID único del Input. Obligatorio para la trazabilidad y el debugging.|

<Aside type="caution" title='Importante'> Por lo tanto, es una **práctica obligatoria** asignar manualmente el atributo `id` a todas las instancias de los inputs para saber identificarlos</Aside>

## Obtención del evento
Para escuchar los eventos emitidos por los componentes de este UI Kit, la opción más confiable es usar **`addEventListener`**, ya que garantiza que los eventos personalizados se reciban **correctamente**.

En **frameworks** _(Vue.js, React, Angular)_ también puede usarse **`addEventListener`**, aunque cada uno ofrece su propia forma de manejar eventos, que igualmente es válida y en la mayoría de los casos funcionan sin problema. Sin embargo, cada framework tiene particularidades importantes que tu como desarrollador debes considerar:

- **Vue**: Normalmente funciona de forma nativa, ya que Vue respeta los **eventos del DOM** tal como se emiten.
- **Angular**: Suele funcionar, en algunas versiones antiguas del framework tenían **limitaciones** con eventos provenientes de Web Components.
- **React**: Puede detectar eventos personalizados pero no siempre. Su sistema interno de `Synthetic Events` no garantiza la **compatibilidad al 100%**, por lo que el comportamiento puede variar entre navegadores y versiones.

export const check = `
<!-- Ejemplo de obtención del evento en cada click -->

<ui-input label="Usuario" placeholder="Tu nombre de usuario" id="input-usuario"></ui-input>

<script>
  /*Se obtiene el input id*/
  const inputControl = document.getElementById('input-usuario');
  // se obtiene el evento 'input' para capturar la entrada en tiempo real
  inputControl.addEventListener('input', (event) => {
    // Valor y el ID
    const { id, value } = event.detail; 
    console.log(\`Input ID: \${id}\`);
    console.log(\`Valor Actual: \${value}\`);
  });
</script>

`;
export const checkm = ['id="input-usuario"', 'addEventListener', 'input-usuario'];

export const rs = `
<!-- Ejemplo de obtención del evento al hacer click en un btn simulando un formulario -->

<ui-input label="Usuario" placeholder="Ingrese su usuario" id="input-usuario"></ui-input>
<ui-button id="btn-submit-form" type="primary">Enviar</ui-button>

<script>
    // Inicio del objeto de datos del formulario 
    let formData = {
        'input-usuario': ''
    };
    const inputUsuario = document.getElementById('input-usuario');
    const btnSubmit = document.getElementById('btn-submit-form');

    // Función para actualizar el dato al perder el foco (evento 'change')
    const updateFormData = (event) => {
        const { id, value } = event.detail;
        if (formData.hasOwnProperty(id)) {
            formData[id] = value;
            console.log(\`Dato actualizado: \${id} = \${value}\`);
        }
    };
    
    /*Escuchar el evento 'change' en el input (se dispara al salir del campo de manera auto)*/

    inputUsuario.addEventListener('change', updateFormData);
    
    /*Escuchar el click en el botón para enviar los datos finales*/

    btnSubmit.addEventListener('click', () => {
        console.log(formData);
        /*Simulación de envío de datos*/
        alert("Dato Obtenido: Usuario = " + formData['input-usuario']);
    });
</script>

`;
export const rsm = ['id="input-usuario"','id="btn-submit-form"', 'addEventListener'];
