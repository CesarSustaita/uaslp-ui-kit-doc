El componente **Tag** `<ui-tag>` _(o Etiqueta)_ es un elemento compacto utilizado para categorizar, filtrar o mostrar metadatos de forma rápida y visual. Se utiliza comúnmente para mostrar estados, categorías de artículos o selecciones activas.

**Vista previa del componente**

<div class="components-structure">
    <div class="responsive-structure"> 
        <ui-tag type="primary" tag="base" >Componente Tag</ui-tag>
        <ui-tag >Componente Tag</ui-tag>
    </div>
</div>

## ¿Cuándo debería usarse?
Usa este componente cuando quieras:

- Categorización: Mostrar etiquetas de artículos _("Noticias", "Deportes")_.
- Estado: Indicar el estado de una entidad _("Activo", "Pendiente")_.
- Filtros: Mostrar filtros aplicados que pueden ser removidos por el usuario.

## Uso básico del componente

El componente se integra directamente en el DOM. Para renderizarlo en el sistema, utiliza la siguiente etiqueta:

export const exampleCode = `<ui-tag> Tag </ui-tag>`;
export const fileName = 'ejemplo.html';

<Code code={exampleCode} lang="js" title={fileName} />

### Atributo `hasBorder`

El atributo booleano `hasBorder` permite añadir un borde a la etiqueta. Su propósito es aumentar la visibilidad de los tags cuando se encuentran sobre fondos que ya son muy claros o blancos.

### Atributo `removable`

El atributo booleano **removable** le da al Tag la funcionalidad interactiva al agregar un **botón de cierre** _("X")_ al final de la etiqueta.

Este atributo es importante para contextos donde el usuario debe gestionar una lista activa de filtros o categorías _(ej. el Tag representa un filtro que el usuario desea eliminar)_.

<Aside type="danger" title="Fundamental">
Es fundamental entender que el componente `<ui-tag>` NUNCA se elimina a sí mismo del DOM.
- **Acción del Componente:** Al hacer clic en la "X", el componente emite un evento.
- **Como Desarrollador**: Es responsabilidad del desarrollador escuchar el evento y ejecutar la acción de eliminar el elemento del DOM o actualizar el estado de la aplicación.
</Aside>

## Disparador de eventos

El componente `<ui-tag>` no gestiona el cambio de estado, sino una acción de notificación al sistema mediante un **Evento Personalizado _(CustomEvent)_** llamado `tag-remove`.

Este evento se dispara cuando el usuario hace clic en el botón de cierre ("X"), notificando a la aplicación que el Tag debe ser eliminado o procesado.

### Estructura de `detail`

| Propiedad | Tipo   | Descripción |
|-----------|--------|-------------|
| `id` | <span style={{whiteSpace: "nowrap"}}>`string` </span>| El ID de la etiqueta marcada para eliminación pendiente. Este valor es crítico para que la lógica del sistema identifique y remueva el Tag correcto. |


<Aside type="danger" title="Práctica Obligatoria:">
Dado que el Tag **NO genera** un ID aleatorio, es obligatorio asignar el atributo **id** manualmente a todas las instancias de **Tags** con el **atributo** removibles (**removable**) para que la aplicación pueda localizarlos y eliminarlos.
</Aside>

## Obtención del evento
Para escuchar los eventos emitidos por los componentes de este UI Kit, la opción más confiable es usar **`addEventListener`**, ya que garantiza que los eventos personalizados se reciban **correctamente**.

En **frameworks** _(Vue.js, React, Angular)_ también puede usarse **`addEventListener`**, aunque cada uno ofrece su propia forma de manejar eventos, que igualmente es válida y en la mayoría de los casos funcionan sin problema. Sin embargo, cada framework tiene particularidades importantes que tu como desarrollador debes considerar:

- **Vue**: Normalmente funciona de forma nativa, ya que Vue respeta los **eventos del DOM** tal como se emiten.
- **Angular**: Suele funcionar, en algunas versiones antiguas del framework tenían **limitaciones** con eventos provenientes de Web Components.
- **React**: Puede detectar eventos personalizados pero no siempre. Su sistema interno de `Synthetic Events` no garantiza la **compatibilidad al 100%**, por lo que el comportamiento puede variar entre navegadores y versiones.


export const js = `
<!-- Ejemplo del Tags Removible en HTML -->
<script>
  // Seleccion del contenedor padre
  const lista = document.getElementById('lista-tags');

  lista.addEventListener('tag-remove', (event) => {
    
    // Extraer el ID del elemento
    const tagId = event.detail.id;

    if (tagId) {
        // Obtener la referencia al elemento Tag
        const tagElement = document.getElementById(tagId);
        
        // Eliminar el Tag del DOM 
        tagElement.remove();
        console.log(\`Tag con ID: \${tagId} ha sido eliminado\`);
    } else {
        console.error("No se pudo eliminar: El Tag no tenía un ID asignado.");
    }
  });
</script>

<div id="lista-tags">
  <ui-tag id="filtro-1" type="primary" tag="negative" removable>Filtro 1 seleccionado</ui-tag>
  <ui-tag id="filtro-2" type="secondary" hasBorder tag="positive" removable>Filtro 2 seleccionado</ui-tag>
  <ui-tag id="filtro-3" type="secondary" tag="base" removable>Filtro 3 seleccionado</ui-tag>
</div>

`;
