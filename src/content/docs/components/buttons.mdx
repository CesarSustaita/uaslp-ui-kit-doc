El **Button** `<ui-button>`es uno de los elementos interactivos más importantes en cualquier interfaz. Su propósito principal es **iniciar una acción**, como enviar un formulario, abrir un modal, confirmar una tarea, navegar a otra vista, etc.

Dentro del sistema de diseño, el botón establece **consistencia**, **jerarquía visual** y **claridad** en las acciones disponibles para el usuario.


**Vista previa del componente**

<div class="components-structure">
    <div class="responsive-structure"> 
        <ui-button type="primary">Button</ui-button>
        <ui-button type="secondary">Button</ui-button>
        <ui-button type="tertiary">Button</ui-button>
    </div>
</div>

### ¿Cuándo debería usarse?

Usa este componente cuando quieras:

- Ejecutar una acción directa (submit, guardar, enviar, continuar).
- Destacar una acción primaria dentro de una pantalla.
- Representar tareas secundarias o alternativas sin perder consistencia.
- Incluir iconos antes o después del texto de la acción.


## Uso básico del componente

El componente se integra directamente en el DOM. Para renderizarlo en el sistema, utiliza la siguiente etiqueta y define el texto de la acción en su contenido interno:

## Uso de Iconos
Los botones están diseñados para integrar iconos en formato **SVG (Scalable Vector Graphics)**, un formato que no contiene pérdida de calidad y permite modificar el color mediante CSS.

El UI Kit utiliza **Slots Nombrados** para controlar la posición del icono, mostrando flexibilidad y variedad en la composición del botón:

### Tipos de integración de iconos

- **Icono al inicio (`slot="icon-start"`)**  
  El icono aparece antes del texto del botón.
  > _Etiqueta del boton: icono + texto = `<ui-button>`_

- **Icono al final (`slot="icon-end"`)**  
  El icono aparece después del texto.
  > _Etiqueta del boton: texto + icono = `<ui-button>`_

- **Botón solo icono (`slot="icon"`)**  
  El botón contiene únicamente el **SVG**, sin texto en su interior.
  > _Etiqueta del boton: solo icono = `<ui-icon-button>`_

<Aside type="tip" title="Conoce más"> Sobre los `slot` consulta la sección "HTML Templates" en "Arquitectura y Tecnología".</Aside>

## Disparador de eventos

El componente `<ui-button>` comunica la interacción del usuario a la aplicación emitiendo un **Evento Personalizado _(CustomEvent)_** llamado `click`.

El evento disparado incluye un objeto `detail`, el cual proporciona información clave para identificar la fuente de la acción.

### Estructura de `detail`

| Propiedad | Tipo   | Descripción |
|-----------|--------|-------------|
| `id`      | string | Identificador único del botón. Esencial para distinguir cuál botón disparó el evento. |


### Manejo del evento

El componente `<ui-button>` (y `<ui-icon-button>`) está diseñado con una **red de seguridad** interna: si el atributo `id` se **omite**, el componente genera un ID único _(string aleatorio)_ para fines de accesibilidad y lógica interna.

Sin embargo, la generación automática de IDs presenta serios inconvenientes en el desarrollo:

- **Inestabilidad en Pruebas:** El ID generado cambia con cada recarga o compilación, haciendo imposible la automatización de pruebas y la depuración.  
- **Falta de Trazabilidad:** Un ID aleatorio no tiene significado semántico, dificultando la lectura y el mantenimiento del código.

<Aside type="caution" title='Importante'> Por lo tanto, es una **práctica obligatoria** asignar manualmente el atributo `id` a todas las instancias de los botones</Aside>

## Obtención del evento
Para escuchar los eventos emitidos por los componentes de este UI Kit, la opción más confiable es usar **`addEventListener`**, ya que garantiza que los eventos personalizados se reciban **correctamente**.

En **frameworks** _(Vue.js, React, Angular)_ también puede usarse **`addEventListener`**, aunque cada uno ofrece su propia forma de manejar eventos, que igualmente es válida y en la mayoría de los casos funcionan sin problema. Sin embargo, cada framework tiene particularidades importantes que tu como desarrollador debes considerar:

- **Vue**: Normalmente funciona de forma nativa, ya que Vue respeta los **eventos del DOM** tal como se emiten.
- **Angular**: Suele funcionar, en algunas versiones antiguas del framework tenían **limitaciones** con eventos provenientes de Web Components.
- **React**: Puede detectar eventos personalizados pero no siempre. Su sistema interno de `Synthetic Events` no garantiza la **compatibilidad al 100%**, por lo que el comportamiento puede variar entre navegadores y versiones.


export const html = `

<script>
  const btn = document.getElementById("btn-enviar");
  btn.addEventListener("click", (event) => {
    console.log("Evento OK:", event.detail.id);
  });
</script>

<ui-button id="btn-enviar" type="positive">Enviar</ui-button>

`;
export const m = ['id="btn-enviar"', 'addEventListener'];
