El componente **Tooltip** `<ui-tooltip>` es un pequeño mensaje de ayuda que aparece cuando el usuario sitúa el puntero sobre un elemento _(hover)_ o navega hasta él con el teclado _(focus)_.

Proporcionar información contextual breve y no esencial que ayude al usuario a comprender la función, el estado o el formato de un elemento de la interfaz sin interrumpir su flujo de trabajo.


**Vista previa del componente**

<div class="components-structure">
    <div class="responsive-structure"> 
        <ui-tooltip text="Este es un Tooltip" >
        <span style="cursor: help;">Implementación de Tooltip haciendo hover en el texto</span>
        </ui-tooltip>
    </div>
</div>

## ¿Cuándo debería usarse?
Usa este componente cuando quieras:

- Aclarar Acrónimos/Términos: Explicar un término técnico o una abreviatura compleja.
- Proporcionar Consejos Breves: Dar una pista rápida sobre cómo usar un campo o un botón (ej. "Solo acepta caracteres alfanuméricos").
- Identificar Iconos: Nombrar una acción representada solo por un icono (ej. el icono de un engranaje).

<Aside type="caution" title='Advertencia de uso'> El Tooltip nunca debe usarse para mostrar información crítica o acciones principales, ya que desaparece fácilmente.</Aside>


## Uso Básico del Componente
El Tooltip funciona como un **Wrapper** ***(envoltorio)*** alrededor del elemento que lo activa. El contenido que el usuario quiere ver se pasa mediante los atributos `text` y `body`.

## Variantes y Atributos
El Tooltip se configura mediante atributos de Contenido y Posición.

**Vista previa del componente**

<div class="components-structure">
    <div class="responsive-structure"> 
        <ui-tooltip text="Texto Principal" body="Cuerpo de texto" id="Ejemplo">
        <span style="cursor: help;">Implementación de Tooltip en posición Top (default)</span>
        </ui-tooltip>
    </div>
</div>


export const tool = `
<!-- Tooltip ejemplo -->
<ui-tooltip text="Texto Principal" body="Cuerpo de texto" id="Ejemplo">
    <span style="cursor: help;">Implementación de Tooltip en posición Top (default)</span>
</ui-tooltip>

`;
export const t = ['text="Texto Principal"', 'body="Cuerpo de texto"', 'id="Ejemplo"' ];

**Aplicación en código**

<Code code={tool} lang="js" title={fileName} mark={t}/>

**Tabla de los atributos implementados**

| Atributo (HTML) | Tipo               | Propósito y Comportamiento |
|------------------|--------------------|-----------------------------|
| `text`           | **string**             | **Contenido Principal.** El mensaje más corto y relevante. Se muestra en negrita como título. |
| `body`           | **string**             | **Contenido Secundario (Opcional).** Texto auxiliar que aparece debajo del mensaje principal. |
| `id`             | **string**             | **Identificación Única.** Necesario si se desea mostrar u ocultar el tooltip mediante JavaScript. |

| Atributo `position` | Dirección de Despliegue                       | Uso Recomendado                                                                 |
|----------------------|-----------------------------------------------|----------------------------------------------------------------------------------|
| `top` (Defecto)      | El Tooltip aparece arriba del elemento.       | Estándar. Ideal cuando hay espacio suficiente por encima del elemento.           |
| `bottom`             | El Tooltip aparece debajo del elemento.       | Recomendado cuando el elemento está cerca de la parte superior (ej. navbar).     |
| `left`               | El Tooltip aparece a la izquierda del elemento.| Útil cuando el espacio a la derecha es limitado.                                |
| `right`              | El Tooltip aparece a la derecha del elemento. | Útil cuando el espacio a la izquierda es limitado.                              |


## Disparador de Eventos

**Comportamiento de Estado (No Emite Eventos)**

El componente Tooltip **NO emite Eventos** Personalizados `CustomEvent` como `tooltip-show` o `tooltip-hide`.

El Tooltip gestiona su propia visibilidad internamente en respuesta a eventos nativos del navegador ***(mouseenter, mouseleave, focus, blur)***. Al no ser un componente de acción o entrada de datos, no es necesario que notifique al sistema padre sobre su cambio de estado.
