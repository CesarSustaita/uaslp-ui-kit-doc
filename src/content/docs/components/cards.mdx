El componente **Card** `<ui-card>` es un contenedor estructural y modular que agrupa contenido relacionado, actuando como un punto de entrada o resumen de información.

**Propósito principal:**

- **Jerarquía** Visual: Delimitar secciones de la interfaz para reducir la carga cognitiva del usuario.
- **Modularidad**: Servir como un espacio reutilizable para listas de artículos, noticias, productos o tareas.
- **Interacción**: Actuar como un elemento clicable que navega a una vista de detalle.

**Vista previa del componente**

<div class="components-structure">
    <div class="responsive-structure-full"> 
        <ui-card imageUrl="/img/Imgcard.svg">
        <span slot="title">Title</span>
        <span slot="description">Body text for whatever you’d like to say. Add main takeaway points, quotes, anecdotes, or even a very short story. </span>
        <div slot="footer">
            <ui-button type="tertiary">Visita el sitio oficial</ui-button>
            <ui-button type="primary">Conoce más</ui-button> 
        </div>
    </ui-card>
    </div>
</div>

## ¿Cuándo debería usarse?
Usa este componente cuando quieras:

- Listar Elementos: Mantener la consistencia en dashboards o galerías _(ej. una lista de trámites o documentos)_.
- Agrupar Contenido: Unir una imagen, un título, una descripción y acciones relacionadas en una sola unidad.
- Navegación Clicable: Proporcionar una área de click para dirigir al usuario a otra página.

## Uso Básico del Componente
El componente utiliza **Slots Nombrados** para estructurar su contenido. La estructura mínima requiere al menos un **título** y una **descripción**, envueltos en sus respectivos slots.

## Variantes
El componente **Card** es altamente personalizable, permitiendo adaptar su orientación y tamaño mediante atributos, y su contenido interno mediante Slots.

**Comportamiento de Dimensiones:** Es importante tener en cuenta cómo el componente maneja el espacio:

- **Altura**: Mantiene una altura consistente basada en su contenido y el atributo `size`.
- **Ancho Fluido (100%)**: _Por defecto_, la tarjeta ocupará el **100% del ancho** disponible de su contenedor.

<Aside type="caution" title='Recomendación de Implementación'>
Para controlar el ancho de la tarjeta, se recomienda colocarla dentro de un contenedor padre que defina las dimensiones ***(como una columna de una grilla CSS o un contenedor con max-width)***. Esto te otorga control total para ajustar la tarjeta al diseño de tu interfaz.
</Aside>

### Atributo de `imageUrl`
Este atributo define la imagen de cabecera (o lateral) de la tarjeta.

- **Opcional**: No es obligatorio definirlo. Si se omite, la tarjeta simplemente no renderizará el área de imagen y ajustará el texto para ocupar el espacio restante.
- **Requisito**: Para su correcta visualización, debes proporcionar una ruta válida ***(relativa o absoluta)*** al recurso de imagen alojado en tu proyecto ***(ej. /img/tarjeta-banner.svg o una URL externa)***.

### Atributo `isInteractive`

El atributo booleano `isInteractive` transforma el comportamiento de la tarjeta, convirtiendo toda su superficie en un elemento activable _("clicable")_.

**Propósito de Diseño** 
Este atributo permite crear interfaces más limpias al eliminar la necesidad de botones explícitos _(como "Ver más" o "Detalles")_ cuando la acción principal es navegar al contenido de la tarjeta.

Comportamiento Visual: Al activar esta propiedad, el componente proporciona estilos hover automáticos al usuario _(para saber que se ha dado click)._

## Disparador de Eventos
El componente `<ui-card>` está diseñado para ser **estático por defecto**. Para habilitar la navegación o la selección, debes activar explícitamente el **atributo** booleano `isInteractive`. 

Cuando una tarjeta interactiva es presionada, emite un Evento Personalizado ***(CustomEvent)*** llamado `card-click`. Este evento está diseñado para manejar la navegación principal _(ej. ir al detalle del producto)_.

<Aside type="tip"> El evento de clic de la `<ui-card>` solo se disparará si este atributo está presente. Si se omite, la tarjeta ignorará los clics del usuario ***(excepto en los botones del footer)***.</Aside>


### Estructura de `detail`
El evento entrega un objeto `detail` con la información necesaria para identificar el elemento seleccionado.

| Propiedad | Tipo          | Descripción                                                                              |
|-----------|---------------|--------------------------------------------------------------------------------------------|
| `id`      | <span style={{whiteSpace: "nowrap"}}>`string` </span>     | El ID único de la tarjeta. Es el dato crítico para saber qué elemento fue seleccionado.|

<Aside type="caution" title='Importante'> Por lo tanto, es una **práctica obligatoria** asignar manualmente el atributo `id` a todas las instancias de los inputs para saber identificarlos</Aside>


## Obtención del evento
Para escuchar los eventos emitidos por los componentes de este UI Kit, la opción más confiable es usar **`addEventListener`**, ya que garantiza que los eventos personalizados se reciban **correctamente**.

En **frameworks** _(Vue.js, React, Angular)_ también puede usarse **`addEventListener`**, aunque cada uno ofrece su propia forma de manejar eventos, que igualmente es válida y en la mayoría de los casos funcionan sin problema. Sin embargo, cada framework tiene particularidades importantes que tu como desarrollador debes considerar:

- **Vue**: Normalmente funciona de forma nativa, ya que Vue respeta los **eventos del DOM** tal como se emiten.
- **Angular**: Suele funcionar, en algunas versiones antiguas del framework tenían **limitaciones** con eventos provenientes de Web Components.
- **React**: Puede detectar eventos personalizados pero no siempre. Su sistema interno de `Synthetic Events` no garantiza la **compatibilidad al 100%**, por lo que el comportamiento puede variar entre navegadores y versiones.
